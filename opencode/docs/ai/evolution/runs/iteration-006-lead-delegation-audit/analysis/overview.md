# Analysis: iteration-006-lead-delegation-audit

The root cause was an unclear boundary between lightweight `/feature` intake and
substantive discovery. When technical/API uncertainty was present, `lead` needed
to build a handoff instead of researching the answer directly.

The fix clarified that:

- initial `lead` analysis is only for scope, uncertainty, and handoff routing;
- technical/product/API/library/risk/architecture uncertainty goes to
  `researcher` before substantive discovery;
- UX/UI/brand/layout uncertainty goes to `designer` before visual discovery;
- `researcher` is not universal for simple, clear features.
