# Analysis: iteration-007-agent-skills-integration

The useful unit of change is local skills plus bounded agent permissions:

- `developer` can use implementation-adjacent process skills;
- `researcher` can use source/security/performance/API/documentation checklists;
- `specifier` can use contract, security, documentation, and testing checklists;
- `reviewer` can use review, testing, debugging, security, and performance
  checklists;
- `lead` continues to route instead of doing implementation work.

The public kit must keep skills generic and free of local providers, MCPs, or
private URLs.
