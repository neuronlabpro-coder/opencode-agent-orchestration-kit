# v1.0.10 - Portable orchestration skills and TUI token totals

This release publishes the latest orchestration improvements from the local
OpenCode harness into the public starter kit, while keeping local providers,
private MCP configuration, credentials, and machine-specific paths out of the
repository.

## Highlights

- Added a portable TUI plugin that shows both lead/root token usage and total
  token usage across child/subagent sessions.
- Added generic local process skills for source-driven development, testing,
  debugging, review, simplification, security, performance, API/interface
  design, and documentation/ADRs.
- Added `/test` and `/code-simplify` as focused auxiliary commands routed
  through `developer`.
- Hardened `lead` so it coordinates and delegates implementation changes instead
  of editing files itself.
- Expanded `lead`'s narrow shell allowlist for routine inspection/navigation:
  `ls`, `find`, `grep`, `rg`, `tail`, `wc`, `cd`, and `which`.
- Clarified `/feature` intake so technical or product uncertainty is delegated
  before broad lead-side discovery.
- Converted AHE evolution records to public-safe English summaries and added
  iterations 005-011 for the newly published harness changes.

## Public-safety notes

- No local provider blocks are included.
- No local MCP configuration is included.
- No credentials, auth files, or private environment variables are included.
- No machine-specific absolute paths are included in `opencode/docs/ai/evolution`.
- The TUI plugin is registered with a relative path in `opencode/tui.json`.

npx tsc --noEmit --jsx preserve --jsxImportSource @opentui/solid --module NodeNext --moduleResolution NodeNext --target ES2022 --strict --skipLibCheck plugins/token-tree-usage.tsx
